# Product

## Register

product

## Users

Engineering teams building **login-based internal tools** on top of this starter — admin panels, dashboards, back-office and operations UIs — plus light public/marketing pages. End users are authenticated operators in a focused task: managing data, monitoring state, configuring settings. They value speed, predictability, and trust over novelty.

## Product Purpose

A production-ready React + TypeScript + Vite SPA starter that ships with correct conventions and a neutral, professional design system, so teams build trustworthy internal tools fast — with AI assistance — without design debt or hallucinated patterns. Success = a new screen looks like it belongs next to Linear/Stripe/Vercel-grade tools on day one, and an AI agent extending it stays on-brand by default.

## Brand Personality

Neutral, professional, trustworthy. Voice: precise, calm, unflashy. Confidence through restraint, not decoration — the interface earns trust by being unsurprising where it should be and considered everywhere. Three words: **precise, calm, dependable.**

## Anti-references

- **Typical AI/SaaS slop** — purple/blurple gradients, default Inter everywhere, endless identical icon+heading+text cards, cream/sand body backgrounds, a tracked-uppercase eyebrow or `01 / 02 / 03` marker above every section.
- **Over-animated / showy** — decorative motion, glassmorphism by default, effects that don't convey state.
- **Dated enterprise admin** — cramped, dull, gray-on-gray back-office screens that feel heavy and untrustworthy.

## Design Principles

1. **The tool disappears into the task.** Earned familiarity beats novelty; reuse standard affordances, don't reinvent them.
2. **Precision builds trust.** Consistent component vocabulary screen to screen; every interactive element ships all its states.
3. **Restraint by default, emphasis by intent.** One accent, used for action/selection/state only — never decoration.
4. **Design every state.** Loading (skeletons), empty (teach the UI), error, disabled — not just the happy path.
5. **Accessibility is a default, not a retrofit.** Meet the bar from the first commit.

## Accessibility & Inclusion

WCAG 2.2 **AA**. Body text contrast ≥ 4.5:1 (≥ 3:1 for large/bold); visible keyboard focus on every interactive element; full keyboard operability; honor `prefers-reduced-motion` with a non-motion alternative for every animation. Don't rely on color alone to convey state (pair with text/icon).
